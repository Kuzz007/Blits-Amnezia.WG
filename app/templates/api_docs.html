Интеграция с Telegram-ботом

Документация API для Telegram-бота

Зачем нужен этот раздел API нужно для автоматизации продаж и поддержки: Telegram-бот или внешний сайт может создавать VPN-клиентов после оплаты, продлевать срок действия, отключать доступ, удалять ключи и получать готовые ссылки/QR без ручного входа в панель.

1. Способы авторизации запросов

Панель поддерживает два полностью защищенных механизма авторизации для Telegram-бота:

Вариант А: Статический API-токен (Рекомендуется) Передавайте этот токен в заголовках HTTP в формате Bearer: Authorization: Bearer {{ telegram_token }}
Вариант Б: Авторизация по логину и паролю панели Бот делает запрос `POST /api/v1/auth/token`, передавая логин и пароль администратора, и получает временный JWT токен, который используется для последующих запросов. Это избавляет от необходимости жестко прописывать токены.

2. Эндпоинты API

POST /api/v1/auth/token

Авторизация бота по логину и паролю панели.

# Пример POST-запроса на получение токена: curl -X POST "http://{{ public_ip }}:{{ panel_port }}/api/v1/auth/token" \ -H "Content-Type: application/json" \ -d '{"username": "admin", "password": "your_password"}' # Ответ сервера при успехе: # { # "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...", # "token_type": "bearer" # }
POST /api/v1/clients

Создать нового пира (клиента) AmneziaWG. Возвращает готовые конфиги и обе версии ссылок импорта.

# Запрос на создание клиента (на 30 дней, без лимита трафика): curl -X POST "http://{{ public_ip }}:{{ panel_port }}/api/v1/clients" \ -H "Authorization: Bearer {{ telegram_token }}" \ -H "Content-Type: application/json" \ -d '{"name": "tg_user_12345", "telegram_id": 12345, "days": 30, "traffic_limit_gb": 0.0}' # Ответ сервера при успехе (возвращает данные и ссылки): # { # "client_id": "a4d3391b-b4be-4161-825f-...", # "name": "tg_user_12345", # "telegram_id": 12345, # "ip_address": "10.66.66.5", # "config_text": "[Interface]\nPrivateKey = ...", # "conf_download_url": "/clients/a4d3391b-b4be-4161-825f-.../download", # "qr_url": "/clients/a4d3391b-b4be-4161-825f-.../qr", # "qr_url_v2": "/clients/a4d3391b-b4be-4161-825f-.../qr?version=2.0", # "qr_url_split": "/clients/a4d3391b-b4be-4161-825f-.../qr?split=true", # "qr_url_split_v2": "/clients/a4d3391b-b4be-4161-825f-.../qr?split=true&version=2.0", # "deep_link": "vpn://...", <-- Amnezia Legacy 1.0 (для старых версий) # "deep_link_v2": "vpn://..." <-- Amnezia 2.0 (для новых версий 4.8.x+) # }
POST /api/v1/clients/{client_id}/extend

Продлить VPN-ключ на указанное число дней (например, после очередной оплаты в боте).

curl -X POST "http://{{ public_ip }}:{{ panel_port }}/api/v1/clients/YOUR_CLIENT_UUID/extend" \ -H "Authorization: Bearer {{ telegram_token }}" \ -H "Content-Type: application/json" \ -d '{"days": 30}'
POST /api/v1/clients/{client_id}/disable

Заблокировать клиента (например, за неуплату). Peer моментально удаляется из активного конфига сервера.

curl -X POST "http://{{ public_ip }}:{{ panel_port }}/api/v1/clients/YOUR_CLIENT_UUID/disable" \ -H "Authorization: Bearer {{ telegram_token }}"
POST /api/v1/clients/{client_id}/enable

Разблокировать ранее отключенного клиента.

curl -X POST "http://{{ public_ip }}:{{ panel_port }}/api/v1/clients/YOUR_CLIENT_UUID/enable" \ -H "Authorization: Bearer {{ telegram_token }}"
DELETE /api/v1/clients/{client_id}

Мягко удалить клиента из системы.

curl -X DELETE "http://{{ public_ip }}:{{ panel_port }}/api/v1/clients/YOUR_CLIENT_UUID" \ -H "Authorization: Bearer {{ telegram_token }}"

3. Пример интеграции бота (aiogram 3.x)

Вы можете прямо скопировать этот класс API-клиента для интеграции в код своего Telegram-бота на Python:

import aiohttp import logging class AmneziaPanelAPI: def __init__(self, base_url: str = "http://{{ public_ip }}:{{ panel_port }}", username: str = "admin", password: str = "your_password"): self.base_url = base_url self.username = username self.password = password self.token = None async def _get_headers(self): if not self.token: await self.login() return { "Authorization": f"Bearer {self.token}", "Content-Type": "application/json" } async def login(self): url = f"{self.base_url}/api/v1/auth/token" payload = {"username": self.username, "password": self.password} async with aiohttp.ClientSession() as session: async with session.post(url, json=payload) as resp: if resp.status == 200: data = await resp.json() self.token = data["access_token"] logging.info("Успешная авторизация в AmneziaWG API!") else: raise Exception(f"Ошибка авторизации API: {resp.status}") async def create_client(self, name: str, telegram_id: int, days: int = 30): url = f"{self.base_url}/api/v1/clients" payload = { "name": name, "telegram_id": telegram_id, "days": days, "traffic_limit_gb": 0.0 } headers = await self._get_headers() async with aiohttp.ClientSession() as session: async with session.post(url, json=payload, headers=headers) as resp: if resp.status == 200: return await resp.json() elif resp.status == 401: self.token = None # Сбросим токен для ре-логина при следующем запросе return await self.create_client(name, telegram_id, days) else: logging.error(f"Не удалось создать пира: {await resp.text()}") return None # Пример вызова в обработчике оплаты Telegram-бота: # api = AmneziaPanelAPI(username="admin", password="password") # client = await api.create_client(name=f"user_{message.from_user.id}", telegram_id=message.from_user.id) # if client: # await message.answer( # f"Оплата принята!
