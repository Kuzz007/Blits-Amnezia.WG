# 🛡️ AmneziaWG VPN Web Panel & Telegram Bot API MVP

Современная, премиальная и безопасная веб-панель на базе **FastAPI**, **SQLite** и **Vanilla CSS** для управления VPN-соединениями AmneziaWG и автоматического выпуска ключей с интеграцией в Telegram-боты.

Проект разработан под **Ubuntu 24.04 LTS**. Все операции маскировки трафика (обфускации) работают на нативном уровне ядра с помощью DKMS-модуля AmneziaWG.

---

## ✨ Основные возможности

1. **Полное управление AmneziaWG на сервере:**
   * Автоматическая сборка DKMS-модуля ядра `amneziawg` из репозиториев разработчиков Amnezia.
   * Настройка NAT (MASQUERADE) маршрутизации для перенаправления клиентского трафика в интернет.
2. **Динамический выпуск ключей:**
   * Выдача текстового конфига (Wireguard-style с Junk-параметрами маскировки).
   * Автоматическая генерация PNG QR-кода для мгновенного сканирования мобильными устройствами.
   * Скачивание готового `.conf` файла.
3. **Бесшовная live-синхронизация:**
   * Добавление/отключение пиров на лету через `awg syncconf` без разрыва активных сессий других пользователей.
4. **Интеграция с Telegram-ботом:**
   * Bearer-auth защищенное API для автоматического создания, продления и блокировки ключей после оплаты.
5. **Премиальный UI-интерфейс (в стиле DashStack):**
   * Полностью адаптивная панель администратора с отображением системных метрик.
   * Безопасное хранение хэшей паролей администратора (bcrypt).
   * Принудительное требование смены стандартного пароля `admin/admin` при первом входе.

## 🚀 Быстрый запуск на сервере в 1 клик (Деплой)

Вы можете развернуть весь проект со своего компьютера на чистый VPS. В проекте настроен автоматический деплой-скрипт.

### Требования на локальном компьютере:
У вас должен быть установлен Python 3.

### Запуск деплоя:
Находясь в папке проекта на своем компьютере, выполните команду:
```bash
python deploy.py
```
Скрипт автоматически:
1. Подключится по SSH к вашему VPS `2.26.0.249`.
2. Загрузит все исходные коды бэкенда, шаблонов и стилей в папку `/root/amnezia-panel-mvp`.
3. Установит необходимые зависимости (`python3-venv`, `pip`).
4. Запустит диагностический скрипт `check_server.sh`.
5. Сформирует виртуальное окружение на VPS и установит библиотеки.
6. Настроит системный сервис автозапуска `/etc/systemd/system/amnezia-panel.service`.
7. Активирует веб-панель в фоновом режиме на порту `8080`.

## 🛠 Ручной запуск и установка на сервере

Если вы хотите запустить проект на сервере вручную:

### 1. Подготовка окружения и скачивание зависимостей:
```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### 2. Запуск панели в ручном режиме:
```bash
uvicorn app.main:app --host 0.0.0.0 --port 8080
```

### 3. Настройка автозапуска через Systemd:
Скопируйте конфигурационный файл службы:
```bash
cp systemd/amnezia-panel.service /etc/systemd/system/amnezia-panel.service
systemctl daemon-reload
systemctl enable amnezia-panel
systemctl start amnezia-panel
```

## 🔒 Безопасность

* **Пароли администраторов:** Хранятся в виде криптографических bcrypt-хэшей. Стандартный пароль `admin/admin` генерируется при первом запуске, и при первом входе веб-интерфейс блокирует доступ к дашборду, принудительно требуя установить новый безопасный пароль на странице `/settings/password`.
* **API токен ботов:** Все запросы к API защищены заголовком `Authorization: Bearer <token>`. Токен авторизации генерируется автоматически при деплое и хранится в приватном файле `.env` на сервере.
* **Сокрытие ключей:** Приватные ключи клиентов никогда не передаются в лог вывода `stdout/stderr` панели.
* **Защита файлов конфигурации:** Файлы клиентов в папке `data/clients/` и папка `/etc/amnezia/amneziawg/` защищены строгими правами доступа `600`/`700` (`chmod` только для владельца-root).

## 📱 Как подключить клиента через приложение AmneziaVPN

Официальное приложение **AmneziaVPN** (доступно на Android, iOS, Windows, macOS) полностью поддерживает протокол AmneziaWG (WireGuard с обфускацией трафика). Импортировать выданный ключ можно четырьмя способами:

1. **Через QR-код (Самый удобный способ):**
   * Откройте приложение AmneziaVPN на смартфоне.
   * Нажмите кнопку **«Добавить сервер»** или знак **«+»**.
   * Выберите вариант **«Сканировать QR-код»**.
   * Направьте камеру смартфона на экран веб-панели с открытым QR-кодом клиента. Приложение само создаст сервер, распознает AmneziaWG протокол и применит настройки обфускации.
2. **Через файл `.conf`:**
   * Скачайте файл конфигурации `<имя_клиента>.conf` из веб-панели.
   * В приложении AmneziaVPN выберите **«Импортировать файл конфигурации»** и укажите скачанный `.conf` файл.
3. **Через ручную вставку текста:**
   * Скопируйте текстовую конфигурацию клиента из веб-панели (кнопка *«Копировать конфиг»*).
   * Откройте AmneziaVPN -> Нажмите **«Настроить самостоятельно»** -> Выберите протокол **«AmneziaWG (OpenVPN/Wireguard с обфускацией)»** -> Вставьте скопированный текст ключа в поле конфигурации.
4. **Deep Link (Будет включен позже):**
   * *Текущий статус:* Deep Link (`vpn://` или `amnezia://`) на этапе MVP отключен (возвращает `null`).
   * *Для его полноценного включения:* Пожалуйста, пришлите образец рабочего экспорта текстового ключа из официального приложения AmneziaVPN (через кнопку *«Share VPN»* -> *«For AmneziaVPN app»*). Наш парсер в `app/deeplink.py` мгновенно расшифрует его формат (обычно это сжатый base64 с json-схемой Amnezia), после чего ссылки заработают в 1 клик на Android и iOS!

## 🤖 Интеграция с Telegram-ботом (REST API)

Все запросы к API должны содержать заголовок авторизации с вашим секретным Bearer токеном из файла `.env` на сервере:
`Authorization: Bearer awg_bot_api_token_71fb589254d3bc7e0da1a2ef490d1bc7`

### 1. Создание нового клиента после оплаты
**Запрос:**
* **Метод:** `POST`
* **Путь:** `/api/v1/clients`
* **Тело запроса (JSON):**
  ```json
  {
    "name": "telegram_123456",
    "telegram_id": 123456,
    "days": 30,
    "traffic_limit_gb": 0
  }
  ```

**Пример CURL запроса:**
```bash
curl -X POST http://2.26.0.249:8080/api/v1/clients \
  -H "Authorization: Bearer awg_bot_api_token_71fb589254d3bc7e0da1a2ef490d1bc7" \
  -H "Content-Type: application/json" \
  -d '{"name": "tg_user_123", "telegram_id": 123456, "days": 30, "traffic_limit_gb": 0}'
```

### 2. Продление ключа клиента
**Запрос:**
* **Метод:** `POST`
* **Путь:** `/api/v1/clients/{client_id}/extend`
* **Тело запроса (JSON):**
  ```json
  {
    "days": 30
  }
  ```

### 3. Временное отключение (блокировка) пира
* **Метод:** `POST`
* **Путь:** `/api/v1/clients/{client_id}/disable`

### 4. Включение ранее заблокированного пира
* **Метод:** `POST`
* **Путь:** `/api/v1/clients/{client_id}/enable`

### 5. Полное (мягкое) удаление клиента
* **Метод:** `DELETE`
* **Путь:** `/api/v1/clients/{client_id}`
*(Пир мгновенно удаляется с сервера AmneziaWG, а в базе данных SQLite выставляется флаг `deleted_at`).*
